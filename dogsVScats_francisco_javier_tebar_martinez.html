Perform the following tasks to prepare the data:

  • Build a numpy array X_train containing one entry for each training image in the data. Each entry must contain the 3-dimensional matrix of pixels corresponding to an image. Make sure to scale down each image to a common size
  • Normalize X_train so that pixels follow a distribution lying in the range [-0.5,0.5]
  • Build a corresponding array Y_train with the class of each training image (0 for cats and 1 for dogs).

  • Randomly shuffle the data. Make sure you shuffle both X_train and Y_train using the same permutation, so you don't lose track of the class of each training pattern.

  • Repeat the whole procedure again to generate X_test and Y_test matrices for the test data. </font>

  • Take advantage of the glob package to generate the filename lists for the indoor and outdoor subfolders.
  • Since you need to repeat the procedure both for training and test data, it would be useful to implement a function that performs all the data loading and transforming steps. </font>

def preprocesing(image, size = 64, mode = "reflect"):
    return(np.array(resize(imread(image), (size, size, 3), mode=mode, anti_aliasing=True)-0.5))

Antes de utilizar estos modelos es necesario tomar algunas consideraciones.
  1. Funcin de prdida
      2. De entre la gran variedad de funciones de prdida disponibles en keras, he elegido binary crossentropy puesto que tenemos un problema de clasificiacin con solamente dos clases. Esta se puede expresar como: $$H_p(q) = -\frac{1}{N_0 + N_1}[\sum_{i=1}^{N_0}log(p(y_i)) + \sum_{i=1}^{N_1}log(p(y_i))]$$ con $N_0$ el nmero de gatos y $N_1$ el nmero de perros.
  2. Solver de optimizacin
      3. En este caso utilizaremos el solver de optimizacin adam ya que funciona bien para problemas con muchos datos y/o parmetros.
  3. Nmero de epochs
      4. Designa el nmero de veces que la red neuronal utilizar todos los datos de entrenamiento para actualizar los pesos. En este caso, de manera un poco manual considero que unas 10 o 20 epochs son suficientes.
  4. Tamao del batch
      5. Designa el tamao del subconjunto de datos tomados de train para actualizar los gradientes. En este caso utilizaremos 128.
  5. Mtrica
      6. Puesto que tenemos un problema con las clases perfectamente balanceadas podemos utilizar una mtrica simple como es accuracy.

    El punto ms importante, con diferencia, a la hora de abordar un problema de clasificacin de imgenes con una red CNN es sin duda la arquitectura a utilizar. En general, considero que es tremendamente complicado construir una buena red partiendo completamente de cero sin tener ninguna referencia. Sobre todo si no disponemos de mucha potencia computacional como es el caso. As, el primer paso es buscar arquitecturas ya creadas que funcionen para tener un punto de partida. Haciendo esto se puede ver que unas de las mejores posibilidades para estos problemas es utilizar un nmero de capas convoluciones mltiplos de 2, en general de 34 para arriba, con kernels de convolucin de 3x3 intercalndolas con MaxPooling. De esta manera, para buscar una configuracin ptima para nuestro problema he partido de una sola CNN con 32 capas convolucionales y posteriormente, he ido aadiendo ms capas teniendo presentes estas consideraciones. Durante este proceso de construir la arquitectura uno se enfrenta a diferentes problemas:

  • El tamao de la imagen a usar
    • En general, no he tenido que pensar mucho en este punto ya que empec con un tamao de 64x64 y al intentar utilizar una resolucin de 128x128 tena problemas de memoria RAM.
  • El nmero de capas densas y la cantidad de dropout utilizado tras el bloque convolucional
    • En este caso es difcil saber cual es el tamao y el nmero de capas ptimo. Las CNNs utilizadas para el problema de 1000 clases empleaban, en general, ms de dos capas densas con una gran cantidad de neuronas con un dropout considerable. Sin embargo, al intentar llevar sto a cabo podan suceder dos cosas: o el modelo hace un overfitting enorme o ms frecuentemente, al emplear un fuerte dropout, la red no aprenda y predeca solamente una de las clases obteniendo un accuracy del 50%. As, en mi caso, para este problema me ha funcionado utilizar una sola capa densa, sobre la cual he ido aumentando el nmero de neuronas en mltiplos de dos desde 16 hasta 512 con el que obtuve los mejores resultados (aunque el overfit es ms pronunciado). Con el dropout tambin he probado y aunque parece no ser tan crtico como el nmero de capas, s es tremendamente til a la hora de evitar el overfit (se puede ver en los resultados de la CNN 1 y CNN 2). Destacar como se ha utilizado GridSearchCV para mostrar como es posible elegir arquitecturas utilizando funcionalidades de sklearn + keras.
  • Estrategia con los bordes de las imgenes
    • En concreto he probado dos estrategias valid y same. La primera es til para redes pequeas ya que con ella se reduce ms rpidamente el tamao de la imagen al pasar por las capas convolucionales al no tratar los bordes, sin embargo preserva mejor el contenido. Mientras que la segunda (same), es mejor cuando trabajamos con tamaos de imgenes pequeos como es nuestro caso (64x64) y un nmero muy grande de capas convolucionales.
  • Elegir las funciones de activacin
    • En general basta con utilizar funciones de activacin Relu para todas las capas, la duda viene a la hora de elegir si utilizar una funcin de activacin softmax o sigmoide en la capa de salida. En mi caso he probado ambas y si bien tampoco suponen una diferencia enorme, la funcin sigmoide da mejores resultados.

    La seleccin de la arquitectura est intrnsecamente ligada a la eleccin del nmero de epochs y el batch size. En este caso, el nmero de epochs se ha establecido a base de prueba y error, utilizando un nmero bajo de epochs y subiendo progresivamente para intentar encontrarnos en un rango entre el overfit y el underfit. No obstante, sto no se puede determinar fcilmente sin utilizar un set de validacin a la vez que se entrena.
